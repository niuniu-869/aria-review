## 1. 目标与受众

**问题**：现在访问 aria-review.com，未登录直接落在登录页——路人不知道这是什么、为什么要注册。

**目标**：一个大气简约的公开 Welcome 页，让三类人各取所需：

| 受众 | 停留时间 | 要带走什么 |
|---|---|---|
| 路人（评委/投资人/同行转发） | 30 秒 | 一句话知道我们做什么 + 看宣传片 |
| 潜在用户（研究者） | 3 分钟 | 理解原理（凭什么可信）+ 想注册 |
| 技术同行 | 5 分钟 | 差异化（≠RAG、写验分离、确定性工程）+ 真实数字 |

**单一职责**：Welcome 页只做"讲清楚 + 引导注册"，不承载任何产品功能。

## 2. 叙事主线（把 5 条碎片拧成一条线）

用户给了 5 条碎片思路：① OpenAI AGI 五层，我们在 Agent→Innovator 之间；② AI4S 都在把实验/回归做得更好，没人问"这个 gap 值得研究吗"；③ 现有综述 agent 只会找文献或塞 RAG，我们能点引用穿透到原文页/表格；④ 愿景是自动捕获领域学术空白；⑤ 0→1 和 1→100 的科研 agent 都有人做了，我们解决"往哪走"。

**一以贯之的主线：「方向」——科研最贵的一步不是做，而是决定往哪做；而一个方向判断，只值它脚下证据的可信度。**

五条碎片在这条线上的位置：

```
 时代坐标(①)  →  版图缺环(⑤②)  →  为什么难+原理(③)  →  真实案例  →  愿景(④)
 L3→L4 之间     0→1/1→100 有人做    方向判断建立在综述上,     MD&A 66 篇   自动捕获
 缺的是"发现     "往哪走"没人做扎实   综述不可信全塌 → 引用     gap 真伪裁决  领域空白
 值得做的事"                        穿透 + 写验分离
```

**贯穿案例**：MD&A（管理层讨论与分析）文献综述——从 66 篇证据矩阵，到两条候选空白的真实裁决（一条验真值得做、一条被判"不是空白是没搜全"）。全页只用这一个案例，所有抽象概念都落在它身上。

**事实核查结论（页面上点名的外部系统，全部属实）**：
- OpenAI 五层框架（2024-07 提出）：Chatbots → Reasoners → **Agents → Innovators** → Organizations。
- 0→1（提出假设/写论文）：Sakana **AI Scientist-v2** 的全 AI 论文通过 ICLR 2025 workshop 同行评审（arXiv:2504.08066）；Google **AI Co-Scientist**（2025-02，Gemini 多智能体假设生成）。
- 1→100（加速执行）：FutureHouse/Edison **Kosmos**（2025-11，单次 12h 自主数据分析，合作者称一次运行≈半年工作量，arXiv:2511.02824）；自动驾驶实验室（self-driving labs）。
- 注意措辞边界：Kosmos 也做"结论挂引用"，所以我们**不声称"只有我们做溯源"**；我们声称的是三件确定性工程（引用存在、证据可溯、日志可离线重算）+ 块级穿透回原文页/表。

- 副题：Aria 把 文献检索 → 全文精读 → 可信综述 → 发现空白 → 验证价值 收进一条**可验证**的研究加速闭环。别的 Agent 给你一个答案，**Aria 给你一条可验证的研究路径**。

### §2 时代坐标：从 Agent 到 Innovator（碎片①）

- 眉题：`01 · 我们在哪`
- 标题：**Agent 已经会做事了。下一层，是发现值得做的事。**
- 正文：OpenAI 在 2024 年提出过一个五层刻度（据报道的内部框架）：会对话的 Chatbots，会推理的 Reasoners，会执行的 Agents，会发现新知识的 Innovators，最后是能运转一个组织的 Organizations。今天的 AI 正站在第三层——它已经能替你跑完一整套任务。但从"会做事"到"会发现值得做的事"，中间隔着一整层。**科研，是这一步跨越最真实的试验场。**（阶梯图角落加 mono 小字出处：`OPENAI · 2024 · 据报道的内部分级`）
- 视觉：五级阶梯图（L1→L5 横向递进），L3→L4 之间的缝隙用朱砂标注 `Aria 在这一步`。

### §3 版图缺环：往哪走，没人回答（碎片⑤②）

- 眉题：`02 · 缺了什么`
- 标题：**从 0 到 1 有人做了，从 1 到 100 有人做了。「往哪走」还没有。**
- 三卡对比：
  - 卡 A `0 → 1 · 提出假设`：AI Scientist-v2 写的论文通过了 ICLR 2025 workshop 同行评审；Google Co-Scientist 能在几天内生成候选假设。**前提：方向已经选好。**
  - 卡 B `1 → 100 · 加速执行`：Kosmos 一次 12 小时运行完成相当于数月的数据分析；自动实验室把实验越做越快、回归越做越准。**前提：方向已经选好。**
  - 卡 C（朱砂描边，视觉重心）`0 之前 · 决定方向`：**这个 gap 真的值得研究吗？** 是真空白，还是你没搜全？值不值得投一年时间？——这是每个研究开始前最贵的判断，也是 AI4S 版图上最后的留白。**Aria 做的就是这一步。**
- 收束句：方向错了，后面跑得越快，浪费越大。

### §4 原理：可信是地基，穿透是证明（碎片③）

- 眉题：`03 · 凭什么可信`
- 标题：**方向判断建立在综述之上。综述不可信，一切归零。**
- 引导段：要判断"这个 gap 值不值得做"，你得先知道"大家都做了什么、还差什么"——这就是综述。可是今天的综述 Agent，要么只能帮你**找到**文献，要么把文献切碎塞进 RAG 或者长上下文，写出一篇**你没法核对**的漂亮文章。一条编造的引用，就足以让整个方向判断作废。
- Aria 的三个回答（三行特性，每行 标题+一句话）：
  1. **结构化精读，不是切块检索** —— MinerU 把整篇文献拆成段、表、图、公式块，结果表格整张保留，Agent 像研究者一样逐块精读，每条抽取都带页/块锚点。
  2. **写验分离** —— 读和抽交给 AI；反查和溯源交给纯代码：cite_check 逐条反查引用、证据哈希绑死来源、RunLog 哈希链可离线重算。
  3. **句句可穿透** —— 综述里的每个引用点一下，直接落到那篇文献的原文段落，连第几页、第几张表都标着。
- **签名交互（全页唯一的"炫技"点）**：内嵌一个可点的「引用穿透」演示——左侧是一段真实综述文字（含 [1] [2] 引用芯片），点击引用，右侧原文面板滑出对应文献的结构块并高亮，锚点标注 `p.7 · 表2`。数据用真实案例数据硬编码，纯 React 状态，无新依赖。面板底部一行小字：`这不是示意图——线上产品里每条引用都这样工作`。

### §5 真实案例：一条 gap 的生与死

- 眉题：`04 · 它真的能跑`
- 标题：**66 篇 MD&A 文献，两条候选空白，一真一假。**
- 叙述（步进式三步，配真实数据）：
  1. **证据矩阵**：66 篇文献每篇一行——研究问题、方法、数据集、主要发现，每格带原文锚点（渲染 data 中 4 行真实矩阵样例为 HTML 表格）。
  2. **空白派生**：从矩阵的张力里派生候选空白（展示 g2：「MD&A 前瞻性语气与盈余管理动机的因果识别仍缺工具变量设计」）。
  3. **价值核验（确定性裁决，无 LLM 参与）**：
     - g2 → 反查命中 2 条 · 得分 0.86 → **verdict: 值得做**（真空白 · 概念断层）
     - g3「可读性指标在中文年报的适用性」→ 反查命中 41 条 · 得分 0.22 → **verdict: 不是空白，是没搜全**
- 收束句：**由证据说话，凭锚点立论**——每一条空白都能回溯到原文。
- 视觉辅助：一张真实产品截图带（workbench / 溯源 / 主题四象限，3 张，横向排列）。

## 4. 视觉设计

### 4.1 设计原则

**大气简约 = 大留白 + 大字号 + 少颜色 + 一处签名交互。** 品牌延续产品既有的「学术宣纸」体系（与登录页、工作台同一血统），不引入新的视觉语言、不引入新依赖、不用任何图库素材——页面上每张图都是真实产品画面。

### 4.2 令牌（全部复用 styles.css 现有 `:root` 变量）

| 角色 | 变量 | 值 |
|---|---|---|
| 底色（浅节） | `--paper` | #faf7f0 |
| 深色节（§2 阶梯 / §6 数字带） | `--indigo` 渐变 | #2f4858 → #1f2a34（同 auth-brand） |
| 强调 / 印章 / CTA | `--cinnabar` | #c0432b |
| 金色点缀（深色节小标题） | `--gold` → #dccaa0 | 沿用 auth-brand-tag |
| 正文墨色 | `--ink / --ink-2 / --ink-3` | 现值 |

### 4.3 字体

- 大标题：`var(--serif)`（宋体系），`clamp(1.9rem, 4.5vw, 3.4rem)`，行高 1.25——学术气质的主要来源。
- 正文：`var(--sans)`，1rem/1.75，测量宽 `max-width: 42em`。
- 眉题/锚点/数字：`var(--mono)`，letter-spacing 0.08em，小号大写——RunLog 的机器感，与"可验证"主题呼应。
- 眉题编号（01–04）成立的理由：页面叙事**真的是一条时间线**（在哪→缺什么→凭什么→跑起来），编号编码的是论证顺序，不是装饰。

### 4.4 签名元素（唯一的 boldness 预算）

**§4 的可点「引用穿透」演示。** 它就是产品本身的杀手锏，在营销页上以真实交互出现——路人不用注册就体验到核心卖点。其余全页保持克制：无渐变彩带、无 3D、无粒子。

### 4.5 动效

- 各节入场：IntersectionObserver 触发一次性 fade+up（12px, 0.5s），`prefers-reduced-motion: reduce` 时全部关闭。
- 引用芯片：hover 微升；点击后原文面板 0.25s 滑入 + 锚点行朱砂高亮脉冲一次。
- 阶梯图：进入视口时 L1→L5 依次点亮（stagger 80ms）。
- 除此以外没有动效。视频不自动播放。

### 4.6 响应式

- 断点 880px（与 auth 一致）：三卡竖排、穿透演示上下堆叠、阶梯图改竖排、截图带横向滚动。
- 视频 16:9 `aspect-ratio` 占位，杜绝布局跳动。

## 7. 部署方案（101.33.32.162）

1. 本地 `pnpm build` → rsync dist 到 `/opt/aria-review/web/`。**若用 `--delete` 必须带 `--exclude 'media/aria-review.mp4'`**，否则每次发版都会删掉服务器上的视频（codex P1）。poster/截图已 commit 进 public/，随 dist 走，无需排除。
2. `rsync aria-review.mp4 → /opt/aria-review/web/media/`（17MB，一次性；之后发版靠 exclude 保留）。
3. nginx：**先在服务器实测再下结论**（repo 内 nginx.conf 的 `try_files` 不能外推到生产配置，codex P2）。验证 `curl /welcome` 是否 200；若无 SPA fallback，给 vhost 补 `try_files $uri /index.html;` 后 reload。
4. 验证：`curl -sI https://aria-review.com/welcome` = 200；`curl -sI …/media/aria-review.mp4` = 200 且 `Accept-Ranges: bytes`（拖动进度条依赖 Range）；浏览器过一遍未登录 `/` 跳转 + 视频可播 + 穿透演示可点。
