# 多源检索 Agent 引擎设计

> 状态: 设计草案 v2 · 2026-07-05 · 已吸收 codex 二审(session 019f30ae)
> 背景: 吸收朋友项目 `yyy-OPS/YFR.frontier-review-daily`(fork 自我们早期 Biblio_Agent) 的**多源检索**能力,补强 Aria Review 的搜索系统。**只缝搜索**,不做其日报/内容运营/微信/打分/变现。

---

## 0. 目标与非目标

**目标**
1. 把搜索从**硬编码 SearchTool** 升级为 **Agent 驱动的多源检索引擎**:Agent 自主决定查哪些源、用什么检索式、是否并发/迭代。
2. 在我们现有双源(OpenAlex 经 R + Sciverse 直连)之外,接入朋友的多源系统(实测可用: CORE + OpenAlex + EuropePMC + Crossref + Unpaywall;随缘: Semantic Scholar / BASE)。
3. **双级筛**: 确定性预过滤收窄 → Agent 按相关性筛 → 人工在候选卡二次确认。**不做 ML/相关性打分**(相关性判断交给 Agent + 人),但**保留确定性规则过滤与稳定排序**(见 §4.4)。
4. 新源候选**仍强制过我们的信任闸门**(安全下载 → MinerU 结构化解析 → cite_check → 溯源),搜索层放开广度,信任闸门原地不动。
5. 多源系统**署朋友名**(NOTICE + 模块 credit)。

**非目标(明确不做)**: 日报/Daily Delta/微信发稿/一图流/水印/翻译/CDK 变现全不做;ML 质量/相关性/新颖性打分不做;定时调度仅留 hook,本期不落。

## 1. 现状对比(为什么是"补强"不是"替换")

| | 我们 · Aria | 朋友 · Frontier |
|---|---|---|
| 源 | 2(OpenAlex 经 R + Sciverse 直连) | 8(实际实现) |
| 模式 | 一次一源,无并发 | 并发多源 + hybrid |
| 排序 | 无,靠上游 + **LLM 自筛** | 三维打分 + novelty |
| 语义检索 | **Sciverse agentic-search(子查询/RAG)** | 无 |
| 全文→溯源 | **content 带页/块锚点 + MinerU 结构化解析** | 只探开放 PDF,无锚点 |
| 接信任内核 | **候选→解析→cite_check→溯源** | 候选→打分→直接喂 LLM |

**结论**: 广度他强、深度我们强。他的是**发现引擎**,我们的是**证据引擎**。缝合姿势 = 用他的发现层补强候选获取,**绝不让其结果绕过信任闸门**。

## 2. 入库"尺子"(录入我们系统的硬要求)

| 层 | 硬要求 | 说明 |
|---|---|---|
| **元数据层**(Paper 题录) | **只有 `title` 必填** | `dedup_key` 由 doi/title 自动算。**注意**: `add_paper` 的 `ON CONFLICT DO NOTHING` + `_fill_missing_fields`(只补空列、非空永不覆盖)是**入库幂等**,不是候选质量合并——见 §4.3 为何不能拿它当跨源 merge。 |
| **溯源层**(可信综述) | **能安全下载到的真 PDF** → MinerU → `content_list`(page_idx+bbox) → `DocumentStructure`(页/块锚点) | MinerU 收 pdf/docx/html(按扩展名判);必须真 PDF 直链且**经安全下载闸门**(§4.5),不是 landing page,不能是内网/超大/登录页。 |

朋友 8 源输出**已归一成我们 Sciverse 的 schema**,故几乎可直接喂现有 `sciverse.py::normalize_meta_result()`(需参数化 source,见 §4.2)。

## 4. 架构设计

### 4.1 分层

```
┌─ Agent 决策层 ────────────────────────────────────────────┐
│  Agent 拿研究问题 → 选 sources[] / query / limit / 迭代      │
│  工具: multi_source_search(sources, query, limit, since)   │
└───────────────┬───────────────────────────────────────────┘
                ▼
┌─ 聚合层(新) ──────────────────────────────────────────────┐
│  1. 并发 fan-out 选定源(只产候选,不写 DB)                   │
│  2. batch 内跨源合并(按 normalized DOI → title+year,择优)   │
│  3. 确定性预过滤 + 稳定排序 + 截断(§4.4)                    │
│  4. 写 ctx["search_candidates"] + emit search_results       │
│     —— 复用现有候选 schema / event / 导入路径(强约束)        │
└───────┬──────────────────────────────┬────────────────────┘
        ▼                              ▼
┌─ 源适配层 ──────────┐        ┌─ 现有双源(行为兼容,不改逻辑)┐
│ PaperSearchClient   │        │ Sciverse(meta+agentic+    │
│ (朋友原创,署名,MIT  │        │  content带页码) / OpenAlex │
│  授权后合入):       │        │ (经 R)                    │
│ CORE/OpenAlex/EPMC/ │        └───────────────────────────┘
│ Crossref/(Semantic/ │
│  HAL/BASE 随缘)     │
└─────────────────────┘
        │入选候选(仅在人确认导入后)
        ▼
┌─ 信任闸门(原地不动,但 PDF 前置见 §4.5) ───────────────────┐
│ 导入→安全下载PDF→MinerU 结构化解析→cite_check→页/块溯源     │
└───────────────────────────────────────────────────────────┘
```

### 4.2 归一化 adapter(必改点)

1. **`normalize_meta_result(row, source="sciverse")` 参数化 source** —— 现 sciverse.py:266-267 硬编码 `source`/`provider`/`externalIds.provider="sciverse"`。改为默认参数 `source="sciverse"` 保持现有 Sciverse 调用**行为兼容**(非字节不动),多源传入各自 source。
2. **EuropePMC 加 `resultType=core`** —— 否则摘要=0、无 PDF(实测证实的朋友漏配)。
3. **CORE venue 空列表 bug** —— `publisher or journals` 当 `journals=[]` 时落成 `'[]'`,需过滤空列表。
4. **provider 命名固定枚举** —— `Paper.source` 仅 40 字符,注释还是 `wos/openalex/arxiv/upload`;多源接入前把 provider 名固定成枚举(openalex/sciverse/crossref/europepmc/core/hal/base/semantic/unpaywall/upload),否则 `source/provider/pdf_source` 会乱。
5. 复用现有 `_dedup_authors`/`parse_year`(已修过 codex P1 误并同姓、float年份→502),8 源候选免费获得这些鲁棒性。

### 4.3 跨源合并在**入库前的聚合层**做(codex P1 纠正)

**不用 DB dedup_key 当跨源 merge。** `add_paper` 的 `ON CONFLICT DO NOTHING` + 只补空字段,是入库幂等;同一批里 Crossref 先入(有DOI无摘要)会挡住 OpenAlex 后到的更优记录(有摘要)。

**正确设计**: 聚合层在**写候选前**做 batch 内合并——
- 合并键: normalized DOI 优先,其次 normalized title+year。
- 择优: 保留所有 `externalIds`;非空字段确定性择优(更长 abstract、更可信 DOI、更全 venue、合并 pdf 候选);记录多源来源(如 `openalex+core`)。
- 只有 Agent/人**选中并导入**后才调 `add_paper`;DB dedup 只做**最终并发幂等保护**,不承担候选质量合并。

### 4.4 确定性预过滤(不是打分,codex P1)

Agent 无法逐条判 500 条(现有 `_SUMMARY_CARD_CAP=150`,10源×50=500 会炸 token)。收窄靠**确定性规则**,不涉及"哪篇更相关"(那仍交给 Agent+人):
- **硬过滤**: 去重、title 非空、年份下限、(可选)has-abstract / has-DOI / has-OA。
- **配额**: 每源上限 + 总候选上限 + summary 摘要截断。
- **稳定排序**: 规则优先级(如年份倒序、来源优先级、有全文优先),**非相关性/质量打分**。
> 措辞对齐: "不做打分" = 不做 ML/加权的相关性或质量分;确定性去重/过滤/稳定排序是工程必需,保留。

### 4.5 PDF 安全下载模块(新,codex P0)——信任闸门前置

`ingest_pdf` 只接受**本地 path**,不负责远程下载。缺一个独立安全模块:
```
resolve_pdf(candidate) -> verified_pdf_asset | reject_reason
```
- **仅对**去重后、过预过滤、**入选**的候选运行(不对全部 500 条跑)。
- **安全下载**: 流式;每跳重定向**重新校验公网 host**(现 `normalize_external_url` 只校验初始 URL,不够);只允许 http/https;拒绝 credential URL / 内网 / `.local`;`content-length` + 实际字节数上限(对齐 `MAX_UPLOAD_BYTES`);超时;防慢速。
- **落盘后校验**: `%PDF` 魔数 + sha256 + 大小;HTML/登录页(200+text/html)拒绝。
- **配额闸门**: MinerU 提交前限批次/页数/候选数,防 fan-out 打爆 OCR 配额。
- Crossref 默认 **metadata-only**,不参与 PDF 探测(实测多为 403/landing)。

### 4.6 Unpaywall 补链(懒加载)

`lookup_unpaywall(doi)`(`/v2/{doi}`,`/search` 端点 500 不用)只对**入选、无 pdf_url、有 DOI** 的候选小批量补;命中率非满(OA 现实),不保证全覆盖。

### 4.7 配置 plumbing(codex P1)

- `Settings` 新增: `core_api_key` / `semantic_scholar_api_key` / `openalex_mailto` / `unpaywall_email` / 每源 timeout·retry·rate-limit。
- **可用源清单检测**: 缺 key 的源**显式标记"未配置"**并对 Agent/前端可见,**不静默 `return []`**(静默会让线上误判"没文献")。

## 5. Agent 化(核心转变)

**现状**: `SearchTool` 硬编码 provider 路由,一次一源。
**目标**: 新增 `multi_source_search(sources|"auto", query, limit, since)` 工具,Agent 自主选源/并发/迭代;**复用**现有 `search_results` event、`ctx["search_candidates"]`、`project__import_search_results`(强约束,防路径分裂)。双级筛: 确定性预过滤 → Agent 按相关性挑 candidate_ids → 人工确认。
**渐进**: 新工具与现有 openalex/sciverse 路径并存(行为兼容),先可选;Agent 就绪后再切默认。

## 7. 分阶段落地

> 每阶段验证 = pytest + 真跑 happy path + codex 二审。**不含 git commit/branch**(项目红线: 用户未要求不主动 git 操作)。

- **M0 · 许可 + 契约**: 拿到 `paper_search_client.py` 书面 MIT 授权;**先定死接口契约**——候选 schema、`resolve_pdf` 安全下载模块签名、配置项、provider 枚举。(阻塞 M2 合入源码)
- **M1 · 源适配 + 配置**: 合入(授权后)源 client + `Settings` 加各源 key/mailto/timeout + 可用源清单检测。修 3 bug(source 参数化 / EuropePMC core / CORE venue)。单测**断言**: EuropePMC 请求含 `resultType=core`;CORE `journals=[]` 不产出 `'[]'`;Unpaywall 只走 `/v2/{doi}`;缺 key 显式"未配置"非静默。
- **M2 · 聚合层 + PDF 安全模块**: 并发 fan-out(只产候选)+ 入库前跨源合并(§4.3)+ 确定性预过滤(§4.4)+ `resolve_pdf` 安全下载(§4.5,SSRF/重定向/大小/配额)。集测: 并发、合并择优、假PDF/HTML/内网拒绝、配额上限。
- **M3 · Agent 工具化**: 暴露 `multi_source_search` + 可用源清单 + 双级筛;复用现有候选/event/导入路径。e2e: Agent 多源检索 → 候选卡 → 人工确认导入。
- **M4 · 溯源贯通**: 入选候选(带 OA PDF)走安全下载 → MinerU → cite_check 端到端,确认页/块溯源不断;Unpaywall 懒加载补链。
- **M5 · 生产复测 + 上线**: 生产机复测 HAL 连通 / CORE 及机构库 PDF 真伪 /(若白名单批了)BASE;上线 20033 / aria-review.com。

## 10. 风险与开放问题

1. **许可 M0 硬前置** —— 未获书面 MIT 授权前不合入朋友源码;先净室 + 定契约。
2. **PDF 安全是承重项** —— SSRF/重定向/配额是 P0,必须在 M2 与聚合同批落地,不能留到 M4。
3. **确定性预过滤 vs "不打分"** —— 需产品确认: 去重/年份/去空/截断/稳定排序不算"打分",是防 token 爆的工程必需(§4.4)。
4. **生产网络** —— 沙箱到 HAL/机构库受限,PDF 层真实上限要生产复测。
5. **Unpaywall 命中率非满 + keywords 8 源全缺** —— 溯源覆盖率非 100%(如实标注);关键词网络会弱(可后补)。
6. **Semantic/BASE 随缘** —— 方案不依赖,配到即启用。
