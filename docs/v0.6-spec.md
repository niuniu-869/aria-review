# BiblioCN v0.6 实施规格（Spec）

> **版本目标**：地基与最小可信闭环
> **周期**：2-3 周（2026-05-21 起）
> **战略口号**：先做"可复现、可追溯、可导出的文献计量综述流水线"
> **审稿**：Claude（初稿） + Codex（二审） 双方共识版，详见 §10

---

## 0 · 范围与原则

### 0.1 做什么

按依赖顺序：
1. **I1 · 统一 markdown 渲染系统**（基础设施，所有后续 AI 输出依赖）
2. **I2 · 用户自带 API key**（基础设施，会话级覆盖 .env 默认值）
3. **N1 · 文献去重 + 元数据修复**（地基，所有 grounding/PRISMA/导出依赖）
4. **F4-pubmed · PubMed 接入**（arXiv 推迟到 v0.7）
5. **F1 · PRISMA 计数 + 流程图**
6. **N2 · AI 引用完整性检查**（轻量校验，比完整 RAG 现实）
7. **F2 · 一键导出综述报告 PDF/DOCX/MD**
8. **F3 · AI Prompt 模板**（精修 3-5 个，不堆数量）

### 0.2 不做什么

- ❌ 综述生成器、引文 grounding 硬约束（v0.7）
- ❌ 引文网络可视化（v0.7）
- ❌ Disruption Index / 思维导图（v0.7）
- ❌ Corpus RAG、本地 Ollama、PaperQA 二次校验、Screening 主动学习（v1.0+）
- ❌ Acwrite 风格"一键成稿"——v0.6 仍以分析驱动为主
- ❌ 持久化用户上传的 key（隐私红线，仅会话内存）

### 0.3 设计原则

- **KISS**：单一职责，能不引入新依赖就不引入
- **YAGNI**：模板库 3-5 个高质量胜过 20+ 个低质量
- **DRY**：markdown 渲染、key 读取、API 调用收敛到单一入口
- **不破坏现有 8 大分析页与 7 个 AI 模块**：新增能力以增量方式接入，旧用户零感知升级

## 1 · I1 · 统一 markdown 渲染系统

### 1.1 现状问题

| 调用点 | 当前实现 | 问题 |
|--------|---------|------|

| `mod_ai_review.R:117` | `<pre>` 显示转义后原文 | 故意不渲染（XSS 顾虑），但可读性差 |
| `mod_ai_summary/translate/screen/rewrite/cite` | `verbatimTextOutput` | 同上，纯文本 |
| `library(markdown)` | 已 `library()` 但无调用 | 未使用 |

### 1.2 目标

- 提供 **唯一** 的 R 函数 `render_markdown_safe(md_text)` → 输出 `shiny::HTML` 可直接喂给 `renderUI`
- **安全 first**：所有 LLM 输出经白名单 sanitizer 过滤，杜绝 XSS
- 支持元素：标题（H1-H6）、段落、粗斜体、列表（有序/无序/嵌套）、链接（`rel=noopener`）、代码块（行内 + 围栏）、表格、引用、分隔线
- 可选元素（按 feature flag 关）：LaTeX 数学（KaTeX）、Mermaid 图、代码高亮（highlight.js）
- 所有 AI 模块输出迁移到统一渲染器

### 1.3 模块设计

**新增文件**：`R/fct_markdown.R`

```r
#' 安全渲染 markdown 文本为 HTML
#'
#' 用 commonmark::markdown_html 转 HTML 后, 经 xml2 + 白名单清洗
#' 防 XSS. LLM 输出场景必经此函数, 禁止直接 HTML().
#'
#' @param md_text character; markdown 原文
#' @param math    logical;   是否启用 LaTeX (KaTeX, 默认 FALSE, v0.7 开)
#' @return shiny::HTML
render_markdown_safe <- function(md_text, math = FALSE) {
  if (!nzchar(md_text)) return(shiny::HTML(""))
  html <- commonmark::markdown_html(md_text,
    extensions = c("table", "strikethrough", "autolink"),
    smart = TRUE)
  html <- sanitize_html(html)          # 白名单清洗, 仅保留 SAFE_TAGS
  if (math) html <- inject_katex(html) # 给 $...$ / $$...$$ 包 <span class=katex>
  shiny::HTML(html)
}

SAFE_TAGS <- c("h1","h2","h3","h4","h5","h6","p","strong","em","del",
               "ul","ol","li","a","code","pre","blockquote","hr","br",
               "table","thead","tbody","tr","th","td","span","div")
SAFE_ATTRS <- list(
  a    = c("href", "title"),
  code = c("class"),
  pre  = c("class"),
  span = c("class")
)
```

```r
#' 获取当前请求该用的 LLM key
#' 优先级: session$userData > .env > NA
#' @param session shiny session
#' @return character 或 NA_character_
resolve_llm_key <- function(session = shiny::getDefaultReactiveDomain()) {
  k <- session$userData$user_api_key %||% NULL
  if (is.null(k) || !nzchar(k))
    k <- Sys.getenv("DEEPSEEK_API_KEY", unset = NA_character_)
  if (is.na(k) || !nzchar(k)) return(NA_character_)
  k
}
```

#### 所有 mod_ai_*.R

`llm_call(...)` 调用处全部增加 `api_key = resolve_llm_key(session)`。

```r
  template <- system.file("report-template.Rmd", package = ...) # 或本地路径
  rmarkdown::render(template,
    output_format = switch(fmt,
      pdf  = rmarkdown::pdf_document(latex_engine = "xelatex"),
      docx = rmarkdown::word_document(),
      md   = rmarkdown::md_document(variant = "gfm")),
    params = list(corpus = corpus, sections = sections, ai = ai_chapters))
}
```

### 7.4 验收标准

## 11 · 风险与回滚

| 风险 | 概率 | 影响 | 缓解 |
|------|-----|------|------|
| `commonmark` 在某些 .nbib 中文场景导致解析异常 | 低 | 渲染失败 | I1 内置 try-catch fallback 到 `<pre>` 显示 |
| Crossref / NCBI 限流 | 中 | 元数据修复慢 | 已用 httr2 retry + throttle |
| PRISMA2020 R 包在 RStudio Server 缺字体 | 中 | PRISMA 图丑 | fallback 到 DiagrammeR + Noto Sans CJK |
| 用户上传 key 安全事故 | **高** | 信任损失 | §2.4 五条铁律全部测试覆盖 + 主动安全审查 |
| PDF 中文乱码 | 中 | 报告无法投稿 | rmarkdown + xelatex + CTeX，CI 加 PDF 视觉回归测试 |

回滚策略：
- 每个功能独立 feature flag（在 `config.yml` 加 `features.v0_6.markdown_render: true` 等）
- 出问题快速切回，不影响其他功能
