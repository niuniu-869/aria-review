真实代码 + 真实日志逐帧拆解 · RESEARCH AGENT

从一句话题目，到检索、综述、找空白、验价值的完整闭环 一个 AI 研究员的完整工作流
而且每一步都查得到出处

它是一条完整的研究流水线：去数据库检索文献 → 写出可信综述 → 找出还没人做的研究空白 → 再二次验证这空白到底值不值得做。最关键的一点：每一步的每一个判断都不靠 AI 拍脑袋，靠的是能指回真实论文、由确定性代码裁决。

一句话讲明白

把它想象成一个极其较真、又讲方法论的研究助理：先去文献库海里捞针找到相关论文；把读过的写成每句都查得到出处的综述；然后站在这些文献的肩膀上指出"哪里还是空白"；最后再反向检索去证伪，确认这个空白不是因为你没搜全才显得空。

I
Sciverse 检索 · search__topic / meta-search / agentic-RAG第一幕 · 找米下锅

综述也好、找空白也好，前提都是"手里得有对的论文"。所以第一步，agent 走 Sciverse（也支持 OpenAlex）去检索文献。关键设计：检索只产出"候选卡"，绝不偷偷往你库里塞东西。挑哪些入库，由你或明确指令决定。

1
检索 · search__topic

设计多个检索式，一次捞一大批候选

agent 的"职责说明书"里写得很死：做综述前的检索不要只查 10 篇，要设计至少 2 个互补检索式，每次 limit 优先用 50、大规模验证用 100（系统综述可上调到 500）。

检索工具 search__topic 支持两个数据源：Sciverse 和 OpenAlex。每条返回的候选都会补一个稳定的 candidate_id（按 外部ID → DOI → 标题 哈希生成），并通过 search_results 事件实时推给前端渲染成候选卡。

打个比方：像在图书馆检索台搜书：系统给你一摞"书目卡片"摊在桌上，但一本都还没借出。你扫一眼标题摘要，挑中的才办借阅（入库）。

2
挑选入库 · project__import_search_results

你挑中的，才"原样"入库，还自动去重

选好后，agent 优先调 project__import_search_results，从最近一次检索的候选批量导入，而不是手工重写一份清单。为什么？因为这样能原样保留工具返回的 DOI、作者、摘要、期刊、URL 和各种外部 ID（Sciverse docId / unique_id / DOI），并用数据库的 DOI/标题自动去重。

  • meta-search：按主题/关键词检索元数据，出候选卡（最常用）
  • agentic-search（RAG）：按语义检索"原文片段（chunk）"，带页码，用于精确取证
  • content：按 docId 拉取某篇全文内容

入库之后，这些论文还会经 MinerU 把 PDF 解析成结构化全文（带页码、块坐标），这正是后面第二/三幕"逐字溯源"的地基。

II
可信综述 · map → reduce → 安全带 → 溯源第二幕 · 可信落笔

有了论文，agent 开始写综述。这一幕的灵魂是：写的每一句话都必须能指回某篇真实论文的某一段原文，编造的引用会被当场抓出来。我们盯着一篇真实论文 [5]（Liu 等 2023《数字金融转型对财务绩效的影响》），看它怎么被读、被引、被溯源。

1
入口 + 大脑 · RunController → step_once

接单，然后 AI 一轮轮自己决定下一步

你点"发送"，后台调度台 RunController 建一条运行记录（状态 running），单线程串行驱动（防止多只手同时写、把账记乱）。

接着进入 agent 循环（每轮叫 step_once）：把对话 + 一份"可用工具清单"交给大模型，大模型要么"再用个工具"、要么"信息够了给最终答复"。读懂你要综述，它就调用 review 工具。

2
精读 Map → 合成 Reduce

先一篇篇读成笔记，再把笔记拼成综述

Map（分而读之）：50 篇论文每篇单独读、6 篇并行，产出结构化"读书卡"（研究问题/方法/发现/关键论断）。关键论断里有一条逐字原文摘录，一段纯代码的定位器会把它精确定位回"第几页第几块"，发一个锚点号。

Reduce（合而成之）：喂给 AI 的是读书卡、不是全文，逼它只能基于读过的东西写。50 篇超过阈值，走分层合成（先分 5 组各写小结，再汇总），每句话后带 [n] 引用。

为什么定位用代码不用 AI？"出处"绝不能让 AI 自己说（它会记错页码）。代码逐字比对，只认完全吻合 / 包含吻合两种铁证，模棱两可的宁可判"没定位上"。

50 篇论文，到底是怎么被读、被写成综述的？

很多人卡在这一步："AI 不就是把 50 篇丢进去让它写吗？"其实不是。一次性塞 50 篇全文，上下文会爆，还会让它东抄西凑地幻觉。实际的做法是先分头读、再分层写，像一个有分工的编辑部。先认清三类角色：

主 agent · 总编

读懂你要综述，决定调 review 工具、定主题、管流程。它不亲自读每一篇、也不亲自写每个字，只编排下面的流水线。

step_once 循环 · 决定调用 review__generate
阅读工 / 写作工 · 专职工人

阅读工：拿到「精读 SOP + 一篇全文」，一次 LLM 调用只读这一篇、产出一张结构化读书卡。写作工：拿到一堆卡片，按 SOP 合成综述。各自专注、互不干扰。

III
GAP 发现 · gap-finder subagent + scratchpad第三幕 · 站上肩膀

综述回答的是"现在大家都研究了什么"；这一幕要回答更难的问题："哪里还没人做"。系统派出一个专门的"找空白小工"（gap-finder subagent），它一边重读论文，一边把发现的研究空白（GAP）记进一块叫 scratchpad 的草稿本。注意：这一幕只发现、不裁决，"这空白值不值得做"留给第四幕。

1
最小授权小工 · dispatch_to_skill

派一个"权限被锁死"的专职小工

主 agent 自己不找空白，只派发一个子 agent 去干这一件事。这个小工被关进"权限笼子"里：它只能用两个工具：read_paper（按需重读原文取逐字证据）和 scratchpad（记空白）。

  • 它没有派发权（不能再叫别的小工）、没有写库权，max_depth=1，被派即到底
  • 权限和限额是写死在代码里的常量，不读磁盘上 SKILL.md 的声明（防止"把文本当安全策略"）
  • 系统只采信它通过 scratchpad 工具结构化提交的内容，不解析它嘴里说的自由文本
打个比方：像请一个临时工只进资料室抄卡片：不给他钥匙串、不让他转包，产出只认他填的标准表格，不听他口头汇报。这样就算他"幻觉"了，也越不了权、糊弄不了系统。
子 agent 权限规格 · SubagentSpec（代码常量）
"gap-finder": SubagentSpec(
  tool_ids = ("read_paper", "scratchpad"),
  # ↑ 仅此两件，绝不含 dispatch（无派发权）
  max_rounds   = 12,
  max_depth    = 1,   # 被父派即到底
  collect_tool_id = ("scratchpad",)
  # ↑ 只信这个工具的结构化输出
)

2
记空白 · scratchpad.add（fail-loud）

每条空白都得"带着证据"才准记下

小工每发现一条空白，就调 scratchpad.add 记一张结构化卡片（GapCandidate）：主题、空白论断（要具体、可证伪）、视角、支撑证据、反证、自评置信度、状态。

最硬的一条规矩：每条空白至少要带 1 条支撑证据，且必须含 paper_id + anchor_id（源坐标）。没有逐字溯源锚点的空白，会被草稿本当场拒收（fail-loud），绝不静默落一条"无凭无据"的空白。

这也顺带完成了你要的"整理现有成果"：每条空白都挂着支撑它的论文（说明现状）和反证论文（说明已被部分覆盖到什么程度），等于把"这块地现在长什么样"一并梳理清楚了。

状态会流转：draft 草稿 → verified 已验证 → accepted 人工采纳（或 rejected）。

IV
价值二次验证 · value-evidence subagent + 确定性 resolver第四幕 · 二次证伪

找到空白还不够，很多"空白"其实是假的：不是没人做，是你没搜全。所以最后一幕专门"打假"：对每条空白做反向检索证伪，再由确定性代码下最终判断。这一步是整套系统的关键设计：判断不由 AI 给，AI 只负责搬证据。

1
攒证小工 · value-evidence subagent

反向检索去"证伪"，只搬证据、绝不下结论

又派一个权限锁死的小工（只能用 read_paper + search + submit_evidence_pack）。它的任务是故意找反对意见：把空白论断转成检索式，去查近年文献，看"声称的空白"是不是其实已经一大堆人做过了。

  • 反向检索（证伪）：命中越多 → 越可能是"伪空白"；命中越少 → 越支持"真空白"
  • 命中去重后算 hit_count（同一篇多个来源不重复计，防虚高）
  • 它把命中的标题/年份/DOI 逐字回传，但绝不写"这是真/伪空白"，那不是它的活

法官 · resolve_value_verdict（纯函数，零 LLM）

一条透明的规则，把证据算成判决

证据齐了，一个纯函数（可单元测试、阈值可配、绝无 LLM 参与）按下面这条公开规则给出三选一的判决：

valuable（真空白） ← 命中 ≤ 3 且 共现网络有结构断层
likely_filled（伪空白） ← 命中 ≥ 25 → 已大量研究，多半是你没搜全
inconclusive（证据不足） ← 其余情况，建议人工复核

所谓"结构断层"，是用第二/三幕已经算好的共现网络（哪些概念常被一起研究）来佐证：如果空白里的两个核心概念在网络里几乎从不相邻，那就是真有一道"结构洞"。判决还附一个 0~1 的透明分数和一句说人话的理由，并永远标注 decided_by="deterministic"。

valuable · 真空白

鲜有研究 + 概念断层 → 值得做。例：g2 命中 2、断层 0.12 → score 0.86。

likely_filled · 伪空白

命中爆表 → 不是空白，是没搜全。例：g3 命中 41 → score 0.22，劝退。

inconclusive · 证据不足

命中不上不下、又无明确断层 → 不硬判，交人工。例：g5 命中 11 → score 留空。

人在环上 · HITL · PATCH gap

最后拍板的是人，而且全程留痕

系统给的是"有证据支撑的判决建议"，不是替你做决定。研究者可以对每条空白 采纳 / 驳回 / 修订（HITL）。每个决策都被 append-only 追加进该空白的运行日志，状态改成 accepted / rejected，可回溯、可验证。

五个 HITL 接口构成闭环：发起发现 → 看草稿本 → 发起核验 → 取判决 → 人工定稿。整条研究链路就此从"检索"一路走到"经过二次验证、人工背书的研究方向"。

为什么整条链都不会胡编 · THE IRON LAWS

支撑这一切的，是四条贯穿始终的"铁律"

把"会不会编造"从概率问题，变成结构问题

  1. AI 只提案，代码裁决。写综述、找空白、攒证据的是 AI；但"有没有出处""是真空白还是伪空白"由确定性代码判定，全程 decided_by=deterministic。
  2. 逐字保留，带源坐标。每条证据都绑论文编号 + 页码 + 块号 + 内容指纹，只认完全/包含吻合的铁证；没有源坐标的空白直接被拒。
  3. 只信工具的结构化输出，不信 AI 的自由发挥。检索结果、空白、证据都来自工具的 ToolResult，绝不解析 AI 嘴里说出的"事实"；子 agent 被最小授权关进权限笼子。
  4. 出错就大声喊（fail-loud），绝不假装成功。核对失败判综述失败、伪造引用如实计数、攒证小工出错就抛错不捏造判决，绝不让没核对过的东西冒充"可信结果"。

所以它给你的，是一条每步都查得到出处、连"值不值得做"都由证据说话的研究闭环。
